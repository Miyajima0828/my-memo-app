# ポート競合回避ガイド

## 概要

Docker環境を構築する際に、既に使用されているポートとの競合を自動的に検出し、空いているポートを見つけて設定するスクリプトを提供しています。

## 自動ポートチェック機能

### 仕組み

1. **ポート使用状況の検出**: `lsof`コマンドを使用して各ポートが使用中かどうかをチェック
2. **空きポートの検索**: 使用中の場合、指定ポートから順に100個のポートをスキャンして空きポートを発見
3. **自動設定**: 見つかった空きポートを`.env`ファイルに自動設定

### 対象ポート

- **アプリケーションポート** (デフォルト: 8000)
  - Nginx Webサーバーがホストに公開するポート

- **MySQLポート** (デフォルト: 3307)
  - MySQLデータベースがホストに公開するポート

- **Viteポート** (デフォルト: 5173)
  - Vite開発サーバーがホストに公開するポート

## 使い方

### 方法1: 自動セットアップ時に実行

`make setup`コマンドを実行すると、自動的にポート競合チェックが実行されます。

```bash
make setup
```

出力例：
```
======================================
  ポート競合チェックスクリプト
======================================

現在の設定:
  - アプリケーション: 8000
  - MySQL: 3307
  - Vite: 5173

ポート使用状況をチェック中...
✗ アプリケーション: ポート 8000 は既に使用されています
→ アプリケーション: ポート 8001 に変更しました
✓ MySQL: ポート 3307 は利用可能です
✓ Vite: ポート 5173 は利用可能です

======================================
.envファイルにポート設定を書き込みます
======================================
✓ .envファイルを更新しました

設定されたポート:
  - アプリケーション: http://localhost:8001
  - MySQL: localhost:3307
  - Vite: http://localhost:5173
```

### 方法2: 手動でポートチェック

既存の環境でポートを確認したい場合：

```bash
make check-ports
```

このコマンドは`.env`ファイルを更新するのみで、Dockerコンテナは起動しません。

### 方法3: スクリプトを直接実行

より詳細な制御が必要な場合：

```bash
bash scripts/check-ports.sh
```

## ポート設定のカスタマイズ

### .envファイルで手動設定

自動検出ではなく、特定のポートを使用したい場合は`.env`ファイルを直接編集できます：

```env
# Docker環境用のポート設定
APP_PORT=8080
MYSQL_PORT=3308
VITE_PORT=5174
```

編集後、コンテナを再起動：

```bash
make down
make up
```

### デフォルトポートの変更

`.env.example`ファイルのデフォルト値を変更することで、プロジェクト全体のデフォルトポートを変更できます：

```env
# Docker環境用のポート設定
APP_PORT=9000
MYSQL_PORT=3308
VITE_PORT=5174
```

## トラブルシューティング

### ポート競合が解決されない

1. **Dockerコンテナの確認**
   ```bash
   docker ps -a
   ```
   他のDockerコンテナが使用している可能性があります。

2. **システムプロセスの確認**
   ```bash
   lsof -i :8000  # 特定のポートを使用しているプロセスを確認
   ```

3. **コンテナの停止**
   ```bash
   make down
   ```

### スクリプトが実行できない

実行権限が付与されているか確認：

```bash
ls -la scripts/check-ports.sh
```

権限がない場合：

```bash
chmod +x scripts/check-ports.sh
```

### macOSでlsofコマンドが使えない

macOSでは通常`lsof`はプリインストールされていますが、使えない場合：

```bash
# Homebrewを使用
brew install lsof
```

### .envファイルが更新されない

1. ファイルの書き込み権限を確認
   ```bash
   ls -la .env
   ```

2. 手動で作成
   ```bash
   cp .env.example .env
   chmod 644 .env
   ```

## 技術詳細

### スクリプトの動作フロー

```
1. .envファイルから現在のポート設定を読み込み
   ↓
2. 各ポートが使用中かチェック（lsofコマンド）
   ↓
3. 使用中の場合、空きポートをスキャン
   ↓
4. 見つかった空きポートを.envファイルに書き込み
   ↓
5. 結果を表示
```

### 使用しているコマンド

- `lsof -Pi :PORT -sTCP:LISTEN`: ポート使用状況の確認
- `sed`: .envファイルの更新（macOS/Linux対応）
- `source`: 環境変数の読み込み

### ポート範囲

各ポートは指定されたデフォルト値から最大100ポート先まで検索します：

- アプリケーション: 8000-8099
- MySQL: 3307-3406
- Vite: 5173-5272

100ポート全て使用中の場合はエラーとなります。

## docker-compose.ymlの環境変数対応

ポート設定は環境変数で制御されており、`docker-compose.yml`内で以下のように定義されています：

```yaml
services:
  nginx:
    ports:
      - "${APP_PORT:-8000}:80"

  mysql:
    ports:
      - "${MYSQL_PORT:-3307}:3306"

  node:
    ports:
      - "${VITE_PORT:-5173}:5173"
```

`${変数名:-デフォルト値}` の形式により、環境変数が未設定の場合はデフォルト値が使用されます。

## よくある質問

### Q: ポートを変更した後、アプリケーションにアクセスできない

A: ブラウザのキャッシュをクリアし、新しいポート番号でアクセスしてください。
   例: `http://localhost:8001`

### Q: データベース接続エラーが出る

A: Laravel側の`.env`ファイルでは`DB_HOST=mysql`のままで問題ありません。
   `DB_PORT=3306`（コンテナ内部のポート）のままにしてください。
   ホストからの接続のみ、変更後のポート（例: 3307）を使用します。

### Q: 複数のプロジェクトで同時に起動できますか？

A: はい、各プロジェクトで異なるポートが自動的に割り当てられるため、同時起動が可能です。

### Q: 本番環境でもこのスクリプトを使うべきですか？

A: 本番環境では固定ポートを使用し、適切にファイアウォール設定を行うことを推奨します。
   このスクリプトは開発環境での利便性向上を目的としています。
